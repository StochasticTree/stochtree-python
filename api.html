StochTree API¶

BART¶

class stochtree.bart.BARTModel¶

Class that handles sampling, storage, and serialization of stochastic forest models like BART, XBART, and Warm-Start BART

predict(covariates: array, basis: array | None = None) → array¶

Predict outcome from every retained forest of a BART sampler.

Parameters:

  • covariates (np.array) – Test set covariates

  • basis (np.array, optional) – Optional test set basis vector, must be provided if the model was trained with a leaf regression basis

Returns:

Array of predictions with as many rows as in covariates and as many columns as retained samples of the algorithm.

Return type:

np.array

sample(X_train: array, y_train: array, basis_train: array | None = None, X_test: array | None = None, basis_test: array | None = None, feature_types: array | None = None, cutpoint_grid_size=100, sigma_leaf: float | None = None, alpha: float = 0.95, beta: float = 2.0, min_samples_leaf: int = 5, nu: float = 3, lamb: float | None = None, a_leaf: float = 3, b_leaf: float | None = None, q: float = 0.9, sigma2: float | None = None, num_trees: int = 200, num_gfr: int = 5, num_burnin: int = 0, num_mcmc: int = 100, sample_sigma_global: bool = True, sample_sigma_leaf: bool = True, random_seed: int = -1, keep_burnin: bool = False, keep_gfr: bool = False) → None¶

Runs a BART sampler on provided training set. Predictions will be cached for the training set and (if provided) the test set. Does not require a leaf regression basis.

Parameters:

  • X_train (np.array) – Training set covariates on which trees may be partitioned

  • y_train (np.array) – Training set outcome

  • basis_train (np.array, optional) – Optional training set basis vector used to define a regression to be run in the leaves of each tree

  • X_test (np.array) – Test set covariates on which trees may be partitioned

  • basis_test (np.array, optional) – Optional test set basis vector used to define a regression to be run in the leaves of each tree. Must be included / omitted consistently (i.e. if basis_train is provided, then basis_test must be provided alongside X_test).

  • feature_types (np.array, optional) – Indicators of feature type (0 = numeric, 1 = ordered categorical, 2 = unordered categorical). If omitted, all covariates are assumed to be numeric.

  • cutpoint_grid_size (int, optional) – Maximum number of cutpoints to consider for each feature. Defaults to 100.

  • sigma_leaf (float, optional) – Scale parameter on the leaf node regression model.

  • alpha (float, optional) – Prior probability of splitting for a tree of depth 0. Tree split prior combines alpha and beta via alpha*(1+node_depth)^-beta.

  • beta (float, optional) – Exponent that decreases split probabilities for nodes of depth > 0. Tree split prior combines alpha and beta via alpha*(1+node_depth)^-beta.

  • min_samples_leaf (int, optional) – Minimum allowable size of a leaf, in terms of training samples. Defaults to 5.

  • nu (float, optional) – Shape parameter in the IG(nu, nu*lambda) global error variance model. Defaults to 3.

  • lambda (float, optional) – Component of the scale parameter in the IG(nu, nu*lambda) global error variance prior. If not specified, this is calibrated as in Sparapani et al (2021).

  • a_leaf (float, optional) – Shape parameter in the IG(a_leaf, b_leaf) leaf node parameter variance model. Defaults to 3.

  • b_leaf (float, optional) – Scale parameter in the IG(a_leaf, b_leaf) leaf node parameter variance model. Calibrated internally as 0.5/num_trees if not set here.

  • q (float, optional) – Quantile used to calibrated lambda as in Sparapani et al (2021). Defaults to 0.9.

  • sigma2 (float, optional) – Starting value of global variance parameter. Calibrated internally as in Sparapani et al (2021) if not set here.

  • num_trees (int, optional) – Number of trees in the ensemble. Defaults to 200.

  • num_gfr (int, optional) – Number of “warm-start” iterations run using the grow-from-root algorithm (He and Hahn, 2021). Defaults to 5.

  • num_burnin (int, optional) – Number of “burn-in” iterations of the MCMC sampler. Defaults to 0.

  • num_mcmc (int, optional) – Number of “retained” iterations of the MCMC sampler. Defaults to 100. If this is set to 0, GFR (XBART) samples will be retained.

  • sample_sigma_global (bool, optional) – Whether or not to update the sigma^2 global error variance parameter based on IG(nu, nu*lambda). Defaults to True.

  • sample_sigma_leaf (bool, optional) – Whether or not to update the tau leaf scale variance parameter based on IG(a_leaf, b_leaf). Cannot (currently) be set to true if basis_train has more than one column. Defaults to True.

  • random_seed (int, optional) – Integer parameterizing the C++ random number generator. If not specified, the C++ random number generator is seeded according to std::random_device.

  • keep_burnin (bool, optional) – Whether or not “burnin” samples should be included in predictions. Defaults to False. Ignored if num_mcmc = 0.

  • keep_gfr (bool, optional) – Whether or not “warm-start” / grow-from-root samples should be included in predictions. Defaults to False. Ignored if num_mcmc = 0.

BCF¶

class stochtree.bcf.BCFModel¶
predict(X: array, Z: array, propensity: array | None = None) → array¶

Predict outcome model components (CATE function and prognostic function) as well as overall outcome for every provided observation. Predicted outcomes are computed as yhat = mu_x + Z*tau_x where mu_x is a sample of the prognostic function and tau_x is a sample of the treatment effect (CATE) function.

Parameters:

  • X (np.array) – Test set covariates

  • Z (np.array) – Test set treatment indicators

  • propensity (np.array, optional) – Optional test set propensities. Must be provided if propensities were provided when .sample() was run.

Returns:

Tuple of arrays with as many rows as in X and as many columns as retained samples of the algorithm. The first entry of the tuple contains conditional average treatment effect (CATE) samples, the second entry contains prognostic effect samples, and the third entry contains outcome prediction samples

Return type:

tuple

predict_tau(X: array, Z: array, propensity: array | None = None) → array¶

Predict CATE function for every provided observation.

Parameters:

  • X (np.array) – Test set covariates

  • Z (np.array) – Test set treatment indicators

  • propensity (np.array, optional) – Optional test set propensities. Must be provided if propensities were provided when .sample() was run and propensity scores were included in the CATE model.

Returns:

Array with as many rows as in X and as many columns as retained samples of the algorithm.

Return type:

sample(X_train: array, Z_train: array, y_train: array, pi_train: array | None = None, X_test: array | None = None, Z_test: array | None = None, pi_test: array | None = None, feature_types: array | None = None, cutpoint_grid_size=100, sigma_leaf_mu: float | None = None, sigma_leaf_tau: float | None = None, alpha_mu: float = 0.95, alpha_tau: float = 0.25, beta_mu: float = 2.0, beta_tau: float = 3.0, min_samples_leaf_mu: int = 5, min_samples_leaf_tau: int = 5, nu: float = 3, lamb: float | None = None, a_leaf_mu: float = 3, a_leaf_tau: float = 3, b_leaf_mu: float | None = None, b_leaf_tau: float | None = None, q: float = 0.9, sigma2: float | None = None, num_trees_mu: int = 200, num_trees_tau: int = 50, num_gfr: int = 5, num_burnin: int = 0, num_mcmc: int = 100, sample_sigma_global: bool = True, sample_sigma_leaf_mu: bool = True, sample_sigma_leaf_tau: bool = False, propensity_covariate: str = 'mu', adaptive_coding: bool = True, b_0: float = -0.5, b_1: float = 0.5, random_seed: int = -1, keep_burnin: bool = False, keep_gfr: bool = False) → None¶

Runs a BCF sampler on provided training set. Outcome predictions and estimates of the prognostic and treatment effect functions will be cached for the training set and (if provided) the test set.

Parameters:

  • X_train (np.array) – Covariates used to split trees in the ensemble. Can be passed as either a matrix or dataframe.

  • Z_train (np.array) – Vector of (continuous or binary) treatment assignments.

  • y_train (np.array) – Outcome to be modeled by the ensemble.

  • pi_train (np.array, optional) – Optional vector of propensity scores. If not provided, this will be estimated from the data.

  • X_test (np.array) – Optional test set of covariates used to define “out of sample” evaluation data.

  • Z_test (np.array, optional) – Optional test set of (continuous or binary) treatment assignments.

  • pi_test (np.array, optional) – Optional test set vector of propensity scores. If not provided (but X_test and Z_test are), this will be estimated from the data.

  • feature_types (np.array, optional) – Indicators of feature type (0 = numeric, 1 = ordered categorical, 2 = unordered categorical). If omitted, all covariates are assumed to be numeric.

  • cutpoint_grid_size (int, optional) – Maximum number of cutpoints to consider for each feature. Defaults to 100.

  • sigma_leaf_mu (float, optional) – Starting value of leaf node scale parameter for the prognostic forest. Calibrated internally as 2/num_trees_mu if not set here.

  • sigma_leaf_tau (float, optional) – Starting value of leaf node scale parameter for the treatment effect forest. Calibrated internally as 1/num_trees_mu if not set here.

  • alpha_mu (float, optional) – Prior probability of splitting for a tree of depth 0 for the prognostic forest. Tree split prior combines alpha_mu and beta_mu via alpha_mu*(1+node_depth)^-beta_mu.

  • alpha_tau (float, optional) – Prior probability of splitting for a tree of depth 0 for the treatment effect forest. Tree split prior combines alpha_tau and beta_tau via alpha_tau*(1+node_depth)^-beta_tau.

  • beta_mu (float, optional) – Exponent that decreases split probabilities for nodes of depth > 0 for the prognostic forest. Tree split prior combines alpha_mu and beta_mu via alpha_mu*(1+node_depth)^-beta_mu.

  • beta_tau (float, optional) – Exponent that decreases split probabilities for nodes of depth > 0 for the treatment effect forest. Tree split prior combines alpha_tau and beta_tau via alpha_tau*(1+node_depth)^-beta_tau.

  • min_samples_leaf_mu (int, optional) – Minimum allowable size of a leaf, in terms of training samples, for the prognostic forest. Defaults to 5.

  • min_samples_leaf_tau (int, optional) – Minimum allowable size of a leaf, in terms of training samples, for the treatment effect forest. Defaults to 5.

  • nu (float, optional) – Shape parameter in the IG(nu, nu*lambda) global error variance model. Defaults to 3.

  • lambda (float, optional) – Component of the scale parameter in the IG(nu, nu*lambda) global error variance prior. If not specified, this is calibrated as in Sparapani et al (2021).

  • a_leaf_mu (float, optional) – Shape parameter in the IG(a_leaf_mu, b_leaf_mu) leaf node parameter variance model for the prognostic forest. Defaults to 3.

  • a_leaf_tau (float, optional) – Shape parameter in the IG(a_leaf_tau, b_leaf_tau) leaf node parameter variance model for the treatment effect forest. Defaults to 3.

  • b_leaf_mu (float, optional) – Scale parameter in the IG(a_leaf_mu, b_leaf_mu) leaf node parameter variance model for the prognostic forest. Calibrated internally as 0.5/num_trees_mu if not set here.

  • b_leaf_tau (float, optional) – Scale parameter in the IG(a_leaf_tau, b_leaf_tau) leaf node parameter variance model for the treatment effect forest. Calibrated internally as 0.5/num_trees_tau if not set here.

  • q (float, optional) – Quantile used to calibrated lambda as in Sparapani et al (2021). Defaults to 0.9.

  • sigma2 (float, optional) – Starting value of global variance parameter. Calibrated internally as in Sparapani et al (2021) if not set here.

  • num_trees_mu (int, optional) – Number of trees in the prognostic forest. Defaults to 200.

  • num_trees (int, optional) – Number of trees in the treatment effect forest. Defaults to 50.

  • num_gfr (int, optional) – Number of “warm-start” iterations run using the grow-from-root algorithm (He and Hahn, 2021). Defaults to 5.

  • num_burnin (int, optional) – Number of “burn-in” iterations of the MCMC sampler. Defaults to 0.

  • num_mcmc (int, optional) – Number of “retained” iterations of the MCMC sampler. Defaults to 100. If this is set to 0, GFR (XBART) samples will be retained.

  • sample_sigma_global (bool, optional) – Whether or not to update the sigma^2 global error variance parameter based on IG(nu, nu*lambda). Defaults to True.

  • sample_sigma_leaf_mu (bool, optional) – Whether or not to update the leaf scale variance parameter based on IG(a_leaf_mu, b_leaf_mu) for the prognostic forest. Defaults to True.

  • sample_sigma_leaf_tau (bool, optional) – Whether or not to update the leaf scale variance parameter based on IG(a_leaf_tau, b_leaf_tau) for the treatment effect forest. Defaults to True.

  • propensity_covariate (string, optional) – Whether to include the propensity score as a covariate in either or both of the forests. Enter “none” for neither, “mu” for the prognostic forest, “tau” for the treatment forest, and “both” for both forests. If this is not “none” and a propensity score is not provided, it will be estimated from (X_train, Z_train) using BARTModel. Defaults to “mu”.

  • adaptive_coding (bool, optional) – Whether or not to use an “adaptive coding” scheme in which a binary treatment variable is not coded manually as (0,1) or (-1,1) but learned via parameters b_0 and b_1 that attach to the outcome model [b_0 (1-Z) + b_1 Z] tau(X). This is ignored when Z is not binary. Defaults to True.

  • b_0 (bool, optional) – Initial value of the “control” group coding parameter. This is ignored when Z is not binary. Default: -0.5.

  • b_1 (bool, optional) – Initial value of the “control” group coding parameter. This is ignored when Z is not binary. Default: 0.5.

  • random_seed (int, optional) – Integer parameterizing the C++ random number generator. If not specified, the C++ random number generator is seeded according to std::random_device.

  • keep_burnin (bool, optional) – Whether or not “burnin” samples should be included in predictions. Defaults to False. Ignored if num_mcmc = 0.

  • keep_gfr (bool, optional) – Whether or not “warm-start” / grow-from-root samples should be included in predictions. Defaults to False. Ignored if num_mcmc = 0.
